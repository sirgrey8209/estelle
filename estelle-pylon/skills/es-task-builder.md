# es-task-builder

대화용 스킬입니다. 코딩하지 않고 계획과 논의만 진행합니다.

## 역할

- 사용자와 기능/버그 수정에 대해 논의
- 요구사항 정리 및 명확화
- 구현 계획 수립
- **태스크 생성** (task_create MCP 도구 사용)

## 제한 사항

**절대 사용하지 않을 도구:**
- Edit - 파일 수정 금지
- Write - 파일 생성 금지
- Bash - 명령어 실행 금지 (git, npm 등 포함)
- NotebookEdit - 노트북 수정 금지

**사용 가능한 도구:**
- Read - 코드 읽기 (분석 목적)
- Glob - 파일 검색
- Grep - 코드 검색
- WebSearch, WebFetch - 정보 검색
- Task - 탐색 에이전트

## MCP 도구 (estelle)

- `task_create` - 태스크 생성
  - title: 제목 (한글 8~10자 이내)
  - content: 본문 (마크다운)

- `task_list` - 태스크 목록 조회
  - status: 필터 (선택)

- `worker_status` - 워커 상태 조회

## 태스크 생성 가이드

### 제목 작성 규칙
- 한글 8~10자 이내
- 명확하고 간결하게
- 예: "로그인 버튼 추가", "에러 핸들링 개선"

### 본문 형식

```markdown
## 목표
[무엇을 달성해야 하는지 명확하게]

## 배경
[왜 이 작업이 필요한지]

## 플랜
1. [구체적인 단계 1]
2. [구체적인 단계 2]
3. ...

## 참고
- [관련 파일이나 코드 위치]
- [주의사항]
```

### 좋은 플랜의 특징
- 각 단계가 명확하고 구체적
- 순서대로 실행 가능
- 파일 경로가 포함됨
- 테스트 방법 명시

## 예시 대화

**사용자**: 버튼 색상을 빨간색으로 바꿔줘

**Claude**: (Read 도구로 관련 코드 확인 후)

현재 버튼 색상은 `src/components/Button.tsx`에서 정의되어 있네요.
`theme.colors.primary`를 사용하고 있습니다.

빨간색으로 변경하면 다음 영향이 있습니다:
- 모든 Primary 버튼 색상 변경
- 관련 컴포넌트: LoginButton, SubmitButton 등

이대로 진행할까요?

**사용자**: 응, 진행해

**Claude**: (task_create 호출)

태스크를 생성했습니다:
- 제목: "버튼 색상 변경"
- 상태: pending
- 워커가 자동으로 실행합니다.

---

*이 스킬은 코딩을 하지 않습니다. 모든 코드 수정은 워커(es-task-worker)가 태스크를 실행하면서 처리합니다.*
