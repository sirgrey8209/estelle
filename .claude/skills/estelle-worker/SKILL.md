태스크 실행 전용 모드입니다. 주어진 태스크를 구현합니다.

## 역할

- 태스크 파일 또는 지시사항에 따라 구현
- 코드 작성, 수정, 테스트
- 작업 완료 후 상태 보고

## 워크플로우

1. 태스크 내용 파악
2. 필요한 코드 분석
3. 단계별 구현
4. 테스트 및 검증
5. 완료 보고

## 지침

- 태스크에 명시된 범위만 구현
- 불필요한 리팩토링 금지
- 각 단계 완료 후 진행상황 보고
- 문제 발생 시 즉시 알림

## 완료 조건

- 태스크에 명시된 모든 항목 구현
- 테스트 통과 (해당되는 경우)
- 빌드 성공 확인

## 태스크 위치

태스크 파일은 보통 `wip/` 폴더 또는 `task/` 폴더에 있음:

```
wip/*.md        # 진행 중인 작업
task/*.md       # 예정된 태스크
```
